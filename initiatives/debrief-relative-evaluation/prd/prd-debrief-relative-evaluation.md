# PRD: 디브리프(상대평가) 기능

## 1. Objective (목표)

디브리프(상대평가) 기능은 라운드의 핵심 미션인 "적합한 인재 채용"을 지원하는 핵심 기능입니다. 제한된 후보자 풀 내에서 가장 적합한 인재를 효과적으로 선별하기 위해, 면접관들이 평가를 종합하고 지원자들을 상대적으로 비교할 수 있는 플랫폼을 제공합니다. 이를 통해 채용 과정의 효율성과 투명성을 높이고, 일관성 있는 의사결정을 지원하여 조직의 인재 경쟁력을 강화하는 것을 목표로 합니다.

---

## 2. Problem Statement (문제 정의)

- 면접관들 간의 평가 의견을 체계적으로 종합하기 어려워, 최종 평가 도출에 많은 시간이 소요되고 있습니다.
- 지원자들 간의 상대적 우수성을 객관적으로 판단하기 어려워, 채용 품질에 영향을 미치고 있습니다.
- 디브리프 과정에서 일관성 있고 효율적인 의사결정 지원이 부족하여, 채용 프로세스의 신뢰성과 투명성이 저하되고 있습니다.
- 현재 면접관들의 참여도와 협업 효율성도 개선이 필요한 상황입니다.

---

## 3. Target Customer & User Stories

### 대상 고객
- 면접관: 개별 평가를 작성하고, 다른 면접관들과 의견을 교환하며 상대평가를 수행하는 사용자
- 채용 담당자/매니저: 모든 면접관의 평가를 종합하여 최종 합격/불합격 결정을 내리는 사용자

### 사용자 스토리

#### 평가 종합 기능
- **면접관으로서** 개별 평가를 작성한 후, 다른 면접관들의 평가를 확인하고 의견을 교환할 수 있어야 합니다.
- **채용 담당자/매니저로서** 모든 면접관의 평가를 한눈에 보고 종합적인 판단을 내릴 수 있어야 합니다.

#### 상대평가 기능
- **면접관으로서** 같은 공고의 다른 지원자들과 비교하여 상대적 우수성을 판단할 수 있어야 합니다.
- **면접관으로서** 사용자가 직접 선택한 지원자들 간의 비교 분석을 수행할 수 있어야 합니다.
- **채용 담당자/매니저로서** 최종 합격/불합격 결정 시 명확한 근거와 함께 결정할 수 있어야 합니다.

#### 디브리프 프로세스
- **면접관으로서** 면접 종료 직후 대면으로 모여 실시간 디브리프를 진행할 수 있어야 합니다.
- **면접관으로서** 서로 다른 시간/장소에서 비동기적으로 디브리프에 참여할 수 있어야 합니다.
- **채용 담당자/매니저로서** 디브리프 진행 상황을 실시간으로 모니터링할 수 있어야 합니다.

---

## 4. Strategic Fit (전략적 적합성)

본 기능은 회사의 미션인 “최고의 인재를 발굴하여 조직의 경쟁력을 강화한다”는 목표와 직접적으로 연계됩니다. 또한, 연간 OKR 중 ‘채용 프로세스 혁신 및 효율성 증대’와 부합하며, 채용 품질 향상과 의사결정 투명성 확보를 통해 조직 전체의 성장에 기여할 것입니다.

---

## 5. What We Believe (핵심 전제)

- 체계적이고 투명한 평가 프로세스가 우수 인재 선발의 핵심이다.
- 면접관 간의 협업과 소통이 평가의 정확성과 신뢰성을 높인다.
- 상대평가를 통한 객관적 비교는 채용 품질을 향상시키는 효과적인 방법이다.
- 실시간 및 비동기 디브리프 지원은 다양한 근무 환경과 일정에 대응하는 필수 기능이다.

---

## 6. Solution Constraints & Principles

### 사용자 인터페이스
- 대시보드를 통해 평가 종합 현황을 직관적으로 제공
- 지원자 비교 뷰는 나란히 비교 가능한 테이블 형태로 구성

### 사용자 경험
- 단계별 안내를 통한 직관적 네비게이션 제공
- 평가 진행 상황과 결과에 대한 명확한 시각적 피드백
- 다양한 사용자 환경을 고려한 접근성 설계

### 시스템 아키텍처 및 기술
- WebSocket 기반 실시간 동기화 지원
- 트랜잭션 처리로 평가 데이터 무결성 보장
- 효율적인 캐싱 전략으로 자주 조회되는 데이터 최적화

### 보안 및 권한 관리
- 역할 기반 접근 제어로 면접관 및 채용 담당자 권한 분리

### 성능 최적화
- 대용량 지원자 및 면접관을 고려한 시스템 확장성 확보

## 7. Narrative (내러티브)

“지금까지 채용팀은 면접관이 개별적으로 남긴 평가를 흩어진 상태로 받아들였다. 최종적으로 누가 적합한 사람인지 결정할 때, 서로 다른 기억과 직관에 의존하며 많은 시간이 소요됐다. 이 과정은 공정하지도, 효율적이지도 않았다.
라운드HR의 디브리프 기능은 이러한 문제를 해결한다. 모든 면접관이 남긴 평가를 한 화면에서 확인하고, 후보자 간 비교 기능을 통해 팀이 합의된 최종 결정을 신속히 내릴 수 있다. 이는 단순한 기능이 아니라, ‘적합한 인재를 채용한다’는 우리의 미션을 직접적으로 실행하는 도구다.”

---

## 8. Functional Requirements

### 8.1 평가 종합 기능
1. 시스템은 면접 종료 후 모든 면접관의 평가를 자동으로 수집해야 합니다.
2. 시스템은 면접관들이 다른 면접관의 평가를 실시간으로 확인할 수 있도록 해야 합니다.
3. 시스템은 이전 면접 평가 이력을 조회할 수 있는 기능을 제공해야 합니다.
4. 시스템은 면접관들 간의 의견 교환을 위한 댓글/토론 기능을 제공해야 합니다.(팀채팅)
5. 시스템은 평가 항목별 점수와 코멘트를 종합하여 시각화해야 합니다.

### 8.2 상대평가 기능
6. 시스템은 사용자가 직접 비교할 지원자들을 선택할 수 있는 기능을 제공해야 합니다.
7. 시스템은 선택된 지원자들의 평가를 나란히 비교할 수 있는 인터페이스를 제공해야 합니다.
8. 시스템은 평가 항목별 상대적 우수성을 시각적으로 표시해야 합니다.
9. 시스템은 지원자별 강점/약점을 비교 분석하여 제공해야 합니다.


### 8.3 디브리프 프로세스 지원
10. 시스템은 실시간 디브리프를 위한 동시 편집 기능을 제공해야 합니다.
11. 시스템은 디브리프 완료 여부를 자동으로 감지하고 채용 담당자에게 알림을 제공해야 합니다.

### 8.4 최종 결과 생성
12. 시스템은 합격/불합격 결정을 위한 투표 기능을 제공해야 합니다.
13. 시스템은 최종 결정에 대한 근거를 자동으로 정리하여 제공해야 합니다.
14. 시스템은 상세한 평가 리포트를 PDF 형태로 생성해야 합니다.
15. 시스템은 평가 리포트에 모든 면접관의 의견과 근거를 포함해야 합니다.

## 9. 솔루션 아키텍처

### 9.1 솔루션 A: 평가 종합 플랫폼 (절대평가)

**핵심 가치 제안:** "지원자의 절대적 가치를 체계적으로 평가"

#### 주요 기능 명세

**A1. 실시간 평가 대시보드**
- 면접관별 평가 진행 현황을 실시간으로 모니터링
- 평가 완료율, 평균 점수, 코멘트 작성 현황을 시각화
- 평가 미완료자에 대한 자동 알림 및 리마인더 기능
- 디브리프 일정 및 참여자 현황 표시
- **디브리프 진행률 대시보드**: 채용 담당자가 실시간으로 디브리프 참여 현황을 모니터링
- **절대평가 현황**: 지원자별 기본정보, 단계별 평가 내역 등 절대적 정보 평가 현황 표시

**A2. 협업 편집 공간**
- 면접관들이 동시에 평가를 작성하고 수정할 수 있는 기능(스코어링 카드)
- 변경 이력 추적 및 버전 관리 시스템
- 실시간 댓글 및 토론 기능으로 의견 교환 지원
- **AI 요약 리포트**: 모든 면접관 평가를 종합하여 지원자별 강점/약점 자동 요약
- **절대평가 템플릿**: 지원자 기본정보, 경력, 기술스택, 면접 성과 등 절대적 평가 기준 제공

**A3. 자동 알림 및 워크플로우**
- 평가 미완료자에 대한 단계별 알림 시스템
- 디브리프 일정 자동 조율 및 참여자 일정 확인
- 평가 완료 시 자동 다음 단계 진행 알림
- 채용 담당자에게 진행 상황 실시간 리포트
- **디브리프 결과 자동 문서화**: 모든 토론과 결정 과정을 자동으로 정리하여 리포트 생성

### 9.2 솔루션 B: 상대평가 엔진 (상대평가)

**핵심 가치 제안:** "같은 조건의 지원자들과 객관적 비교 평가"

#### 주요 기능 명세

**B1. 지원자 간 비교 인터페이스**
- 같은 공고 & 같은 단계 내 선택된 지원자들의 평가를 나란히 비교할 수 있는 테이블 형태
    * Advance : 같은 공고의 같은 단계의 평가 내역이 있는 모든 지원자 선택
        eg. 2차 면접 단계 비교 중 - 현재 불합격 단계에 있지만 2차 면접 단계의 평가 내역이 있는 경우 비교 가능
- 평가 항목별 점수, 코멘트, 강점/약점을 한눈에 비교
- 사용자가 직접 비교할 지원자들을 선택할 수 있는 기능
- 선택된 지원자들의 평가 결과를 나란히 비교 분석
- 비교 결과를 시각적으로 강조하는 UI/UX
- 포지션별, 부서별, 경력별 세분화된 그룹핑 옵션
- 그룹 내 지원자 수 및 평가 현황 요약 제공
- 평가 항목별 상대적 우수성을 시각적으로 표시
- 면접관 의견을 기반으로 한 정성적 분석 결과 제공
- 조직 문화 적합도 및 기술 적합도 종합 분석(AI 이력서 스크리닝 결과)


### 9.3 통합 아키텍처

**완전한 디브리프 솔루션 아키텍처:**

```
면접 완료
    ↓
디브리프 세션 시작
    ↓
솔루션 A: 지원자 종합 평가(절대평가)
    ├── A1. 평가 대시보드 (스코어링 카드)
    └── A2. AI 요약 리포트 
    ↓
솔루션 B: 디브리프 페이지 (상대평가)
    └── B1. 지원자간 비교 인터페이스
    ↓
절대평가 + 상대평가 통합 분석
    ↓
최종 평가 결과 도출
```
**개념 정의**
1. A1. 평가 대시보드 : 해당 평가의 모든 면접관 최종 평가 점수를 보여주는 스코어링 보드(ref. brighthire 스코어링 카드) 
2. A2. AI 요약 리포트 : 지원자 정보 + AI 평가 요약 + 근거 하이라이트
3. B1. 지원자간 비교 인터페이스 : 같은 공고, 같은 단계의 지원자끼리 종합 평가 결과를 비교한다(지원자 카드 묶어서 보여주기)
**데이터 플로우:**
1. 면접 완료 → 디브리프 세션 시작
2. 디브리프 진행 → 솔루션 A(절대평가)와 솔루션 B(상대평가) 동시 활용(뷰 옵션으로 제공)
3. 절대평가: 지원자 기본정보, 단계별 평가 내역 등 절대적 정보 평가
4. 상대평가: 같은 공고의 같은 단계 지원자들과의 상대적 비교 평가
5. 통합 분석 → 절대평가와 상대평가 결과를 종합하여 최종 평가 도출
6. 결정 완료 → 자동 문서화 및 다음 단계 진행

**사용자 여정:**
- **면접관**: 개별 평가 작성 → 절대평가(A)와 상대평가(B) 동시 확인 → 통합 분석 결과 검토 → 최종 의견 제시
    - 절대 평가 뷰에서 다른 면접관의 평가 정보를 확인할 수 있다.
    - 상대 평가 뷰에서 다른 지원자의 평가 정보를 동시에 비교할 수 있다.
    - 최종적으로 나의 평가 결과(점수)를 수정하거나 확정할 수 있다.
- **채용 담당자**: 전체 현황 모니터링 → 절대평가/상대평가 결과 종합 검토 → AI 인사이트 활용 → 최종 결정 → 자동 문서화
    - 모든 면접관이 평가 결과를 확정하면 최종 평가 결과가 나온다
    - 최종 평가 결과를 바탕으로 다음 진행 여부를 결정한다.

## 10. Out of Scope

1. **AI 기반 자동 평가**: AI가 면접관을 대신하여 평가를 수행하는 기능  
2. **실시간 면접 지원**: 면접 진행 중 실시간 지원 기능  
3. **B4. 추천 순위 시스템**
    - 상대평가 결과를 기반으로 한 자동 순위 제안
    - 가중치 적용 가능한 평가 기준 설정 기능
    - 최종 합격/불합격 결정을 위한 명확한 근거 제공
