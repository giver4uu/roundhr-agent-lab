# 보도자료(Press Release): 라운드HR 디브리프(상대평가) 기능 출시

발행일: 2025년 1월 15일  
발행처: 라운드HR 제품팀

## 헤드라인
라운드HR, 면접 후 합의 시간을 절반으로 줄이는 디브리프(상대평가) 기능 출시

## 서브헤드라인
절대평가 대시보드와 지원자 간 비교 인터페이스를 통합해 면접관들이 빠르고 일관된 채용 결정을 내리도록 지원합니다.

## 본문
서울 — 라운드HR는 오늘, 면접 이후 팀이 신속하고 투명하게 최종 결정을 내릴 수 있도록 돕는 ‘디브리프(상대평가) 기능’을 정식 출시한다고 발표했습니다. 디브리프 기능은 면접관별 평가를 종합하는 절대평가 대시보드와, 같은 공고·같은 단계의 지원자를 나란히 비교하는 상대평가 인터페이스를 하나의 흐름으로 제공하여, 평균 디브리프 완료 시간을 최대 50%까지 단축하도록 설계되었습니다.

디브리프 기능은 면접관들이 작성한 점수와 코멘트를 한 화면에서 시각화하고, 비교가 필요한 후보들을 선택해 항목별 강점/약점을 직관적으로 파악할 수 있는 UI를 제공합니다. 또한 비동기 디브리프 진행상태 관리와 투표 기능을 통해, 시공간 제약 없이 합의 과정을 완결할 수 있습니다.

## 고객 가치
- 시간 절감: 디브리프 완료 시간을 기존 대비 50% 단축(목표)  
- 의사결정 일관성: 면접관 간 점수 표준편차 20% 감소(목표)  
- 투명성 강화: 평가 근거 문서화율 100%(목표)  
- 협업 효율: 비동기 디브리프 완료율 95%(목표)

## 고객 인용문(예시)
“그동안 디브리프는 기억과 인상에 좌우되기 쉬웠습니다. 라운드HR의 비교 인터페이스 덕분에 팀이 같은 정보를 보며 신속하게 합의할 수 있었습니다. 회의 시간이 절반으로 줄었습니다.”  
— 김OO, 테크 리크루팅 헤드, 스케일업 A사

## 작동 방식(How it works)
1) 면접 종료 시, 라운드HR은 면접관들의 평가(점수·코멘트)를 자동 수집해 절대평가 대시보드에 시각화합니다.  
2) 팀은 디브리프 세션에서 같은 공고·같은 단계의 지원자를 선택해 항목별로 나란히 비교합니다.  
3) 비동기 상태관리와 투표 기능으로 합의 과정을 마무리하면, 결과와 근거가 자동 문서화되어 다음 채용 단계로 전달됩니다.

## 출시 범위(MVP)
- 절대평가 대시보드(스코어링 카드)  
- 지원자 간 비교 인터페이스(수동 선택, 동일 공고·동일 단계 기준)  
- 비동기 디브리프 진행상태 관리, 투표, 근거 자동 정리(텍스트)  
- 권한 분리(면접관/채용담당) 및 기본 감사 로그

참고: 기존 라운드의 AI 이력서 스크리닝과 AI 요약은 의사결정 보조로 제공됩니다(자동 판정/점수 산출 제외). 추천 순위 자동화는 후속 릴리스 대상입니다.

## 가용성 및 가격
- 일반 가용성: 2025년 2분기(파일럿 고객 우선 제공 후 순차 확대)  
- 가격: 기존 라운드HR 플랜에 포함(엔터프라이즈 플랜의 경우 고급 권한/감사 옵션 제공)

## 성공 지표(요약)
- 디브리프 완료 시간 50% 단축  
- 평가 점수 표준편차 20% 감소  
- 문서화율 100%, 비동기 완료율 95%  
- 사용자 만족도 4.5/5.0 이상

# FAQ: 라운드HR 디브리프(상대평가)

## Q1. 기존 평가 도구와 무엇이 다릅니까?
디브리프는 면접 직후 “합의 과정”에 초점을 맞춘 기능으로, 절대평가(개별 후보의 종합 정보)와 상대평가(후보 간 직접 비교)를 하나의 여정으로 통합하여 의사결정을 빠르고 일관되게 만듭니다.

## Q2. 비교 가능한 후보의 범위는 어떻게 정의되나요?
같은 공고·같은 단계에서 평가 내역이 존재하는 후보를 기본 그룹으로 자동 제안하며, 사용자는 이 그룹 내에서 비교 대상을 수동 선택합니다. 상태(합격/보류/불합격)와 무관하게 해당 단계 평가가 존재하면 비교 가능합니다.

## Q3. AI 기능은 무엇을 하며, 무엇을 하지 않나요?
AI 이력서 스크리닝 및 AI 요약은 정보를 보조적으로 요약·하이라이트해 의사결정을 돕습니다. 다만 AI가 최종 점수/판정을 자동 산출하지는 않습니다(자동 판정은 범위 외). 추천 순위 자동화는 후속 버전에서 고려됩니다.

## Q4. 팀 합의는 어떻게 이뤄지나요?
비동기 진행상태 보드에서 토론과 코멘트를 남기고, 최종 투표로 결정을 확정합니다. 모든 변경·결정은 감사 로그에 기록됩니다.

## Q5. 데이터 보안과 권한 관리는 어떻게 되나요?
역할 기반 접근제어(RBAC)로 면접관/채용담당/관리자의 권한을 분리합니다. 평가 데이터는 암호화 저장되며, 열람·수정·확정 이벤트는 모두 감사 로그로 추적됩니다.

## Q6. 성능 목표는 무엇인가요?
비교 뷰 p95 로딩 2초 이하(4명·10항목 기준), 평가 제출 p95 800ms 이하, 상태 반영 지연 3초 이내(준실시간) 목표를 권장합니다.

## Q7. 오프라인/실시간 충돌은 어떻게 처리하나요?
MVP에서는 오프라인 편집은 제한하고, 연결 복구 시 제출·상태 동기화 위주로 지원합니다. 동시편집 충돌 관리는 후속 버전에서 강화합니다.

## Q8. 가중치 기반 종합 점수는 누가 어떻게 관리하나요?
조직 정책 또는 채용담당 권한으로 가중치를 설정·변경하고, 변경 이력은 리포트와 감사 로그에 표시됩니다.

## Q9. 어떤 지표로 효과를 측정하나요?
디브리프 완료 시간, 점수 표준편차, 비동기 완료율, 사용자 만족도, 문서화율을 핵심 지표로 추적합니다. 이벤트(제출, 비교 열람, 투표, 확정, 내보내기)를 계측합니다.

## Q10. 롤아웃 계획은 어떻게 되나요?
파일럿(5~10개 팀) → 기능 안정화 → 일반 가용성 순으로 단계적 확대합니다. 파일럿 기간 동안 성능·권한·UX 가드레일을 모니터링합니다.

## Q11. 기존 시스템과의 통합은?
라운드HR 내 기존 평가/후보 데이터와 네이티브 통합됩니다. 외부 ATS/캘린더 연동은 단계적으로 지원합니다.

## Q12. PDF 리포트는 제공되나요?
MVP에서는 텍스트 기반 근거 정리를 제공하며, PDF 내보내기는 후속 릴리스에 포함될 예정입니다.

## Q13. 편향 방지는 어떻게 고려하나요?
면접관별/항목별 분포 시각화, 익명화 옵션(조직 정책에 따라), 가중치 프로파일 가이드 등으로 판단 편향을 낮추도록 설계합니다.
