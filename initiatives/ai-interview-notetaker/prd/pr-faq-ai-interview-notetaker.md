### 본문

**서울, 대한민국** - 대화형 채용 플랫폼 라운드가 오늘 'AI 면접 노트테이커' 기능을 출시한다고 발표했습니다. 이 기능은 면접관이 메모 작성에 신경 쓰지 않고 온전히 지원자와의 대화에 집중할 수 있도록 지원하며, 동시에 면접 내용을 구조화된 데이터로 변환하여 더욱 객관적이고 정확한 채용 의사결정을 가능하게 합니다.

#### 면접관과 지원자, 모두가 만족하는 면접 경험

많은 면접관들이 면접 중 지원자의 답변을 필기하느라 대화의 흐름을 놓치거나, 지원자의 미묘한 표현이나 태도를 관찰하지 못하는 어려움을 겪어왔습니다. 라운드의 AI 면접 노트테이커는 이러한 문제를 근본적으로 해결합니다.

"면접관이 노트북이나 메모지만 바라보고 있으면 지원자는 자신이 진정으로 존중받고 있다고 느끼기 어렵습니다," 라운드의 제품 책임자는 말합니다. "AI가 자동으로 모든 내용을 기록하고 분석하면, 면접관은 지원자의 눈을 바라보며 깊이있는 대화를 나눌 수 있습니다. 이것이 진정한 채용 경험의 혁신입니다."

#### 주요 기능

**1. 실시간 녹음 및 전사 (Must-have, MVP)**
- Zoom, Google Meet 등 주요 화상회의 플랫폼에 AI 노트테이커가 자동 참여
- OpenAI Whisper 기술을 활용한 정확한 한국어/영어 실시간 전사
- 지원자 동의 프로세스 자동화로 법적 컴플라이언스 보장

**2. AI 기반 요약 및 분석 (Must-have, MVP)**
- 면접 종료 즉시 AI가 핵심 내용을 구조화된 리포트로 요약
- 지원자의 강점, 약점, 주요 경험, 역량 증거 자동 추출
- 면접관의 주관적 기억이 아닌 실제 대화 내용 기반 객관적 평가 지원

**3. 실시간 추가 질문 제안 (Must-have, MVP)**
- 지원자의 답변을 실시간으로 분석하여 추가 역량 검증 질문 제안
- "리더십을 더 깊이 검증하려면 이런 질문을 해보세요" 식의 실시간 코칭
- 면접관의 경험 수준과 무관하게 일관된 고품질 면접 수행 가능

**4. 구조화된 평가 리포트 (Must-have, MVP)**
- 직무 요구사항 대비 지원자 역량 매핑
- 면접 질의응답을 토대로 한 객관적 분석
- 디브리프 미팅에서 활용 가능한 증거 기반 평가 자료

**5. 라운드 Conversational ATS 통합**
- Slack에서 "이번 김철수 면접 요약해줘" 명령으로 즉시 요약 확인
- 기존 면접 일정 → 피드백 수집 워크플로우에 자연스럽게 통합
- 팀 전체가 면접 녹음 및 전사 내용 공유 가능 (권한 설정)

#### 라운드만의 차별점: 온톨로지 지식 그래프

라운드의 AI 면접 노트테이커가 단순한 녹음 도구와 다른 점은 **온톨로지 지식 그래프** 기술을 활용한다는 것입니다. 면접 대화 내용은 단순히 텍스트로 저장되는 것이 아니라, 구조화된 지식으로 변환됩니다.

예를 들어:
- "김철수 지원자가 언급한 프로젝트 관리 경험" → 프로젝트 관리 역량 노드
- "5명 규모 팀 리딩 경험 2년" → 리더십 역량 노드
- "애자일 스프린트 운영 경험" → 애자일 방법론 역량 노드

이러한 구조화된 데이터는:
1. **채용 검증의 질적 향상**: 직무 요구사항과 지원자 역량을 정밀하게 매핑
2. **학습하는 채용 시스템**: 성공한 채용과 실패한 채용의 패턴을 학습하여 더 나은 질문 제안
3. **조직의 채용 지식 축적**: 면접 대화 데이터가 쌓일수록 더 정확한 인사이트 제공

"정적인 이력서 데이터만으로는 지원자의 진짜 역량을 파악하기 어렵습니다," 라운드 대표는 설명합니다. "면접에서 나눈 동적인 대화 데이터를 확보하고 구조화하면, 우리는 훨씬 더 깊이있는 채용 인텔리전스를 제공할 수 있습니다."

## FAQ

### 제품 및 기능 관련

#### Q1. 기존 피드백 수집 기능과 무엇이 다른가요?

**A1.** 기존 피드백 수집은 면접관이 면접 후 **기억에 의존하여** 주관적으로 작성하는 방식입니다. AI 면접 노트테이커는:

1. **객관적 기록**: 실제 면접 대화 전체를 녹음/전사하여 "지원자가 정확히 무엇을 말했는지" 기록
2. **실시간 지원**: 면접 중 추가 질문을 제안하여 면접 품질 자체를 향상
3. **자동 분석**: AI가 면접 내용을 분석하여 구조화된 리포트 제공
4. **증거 기반 평가**: "지원자가 리더십이 부족하다"는 주관적 느낌이 아니라 "리더십 경험에 대한 질문에 구체적 사례를 제시하지 못했다"는 객관적 사실 기반 평가

기존 피드백 수집과 **대체가 아닌 보완** 관계입니다. AI 노트테이커가 초안을 제공하면 면접관이 최종 검토/수정합니다.

#### Q2. 실시간 추가 질문 제안은 어떻게 작동하나요?

**A2.** 면접 중 지원자의 답변을 AI가 실시간으로 분석하여:

1. **역량 검증 깊이 평가**: "이 답변으로 리더십 역량이 충분히 검증되었는가?"
2. **후속 질문 제안**: Slack 또는 면접관 대시보드에 "이런 질문을 추가로 해보세요" 알림
3. **면접관 선택**: 제안을 채택할지 무시할지는 면접관이 결정 (AI는 조력자)

예시:
- 지원자: "팀 프로젝트를 성공적으로 완료했습니다"
- AI 제안: "구체적으로 어떤 역할을 하셨는지 물어보세요" / "갈등 상황은 없었는지 확인해보세요"

MVP에서는 **면접 중 실시간 제안**을 제공하며, Phase 2에서는 **역량 갭 자동 감지** (예: "협업 역량이 아직 검증되지 않았습니다")로 고도화됩니다.

#### Q3. 온톨로지 지식 그래프는 구체적으로 어떻게 활용되나요?

**A3.** 온톨로지 지식 그래프는 라운드만의 핵심 차별점입니다. 면접 대화를 단순한 텍스트가 아닌 **구조화된 지식**으로 변환합니다:

**데이터 구조 예시:**
```
지원자(김철수)
  ├─ 언급한 역량
  │   ├─ 프로젝트 관리 (경험: 2년, 규모: 5명 팀)
  │   ├─ 리더십 (경험: 팀 리드 1회, 멘토링 2명)
  │   └─ 기술 스택 (React, Node.js, AWS)
  ├─ 검증된 역량
  │   ├─ 프로젝트 관리 (검증도: 높음, 근거: 구체적 사례 3개)
  │   └─ 리더십 (검증도: 중간, 근거: 멘토링 사례 1개)
  └─ 미검증 역량
      └─ 갈등 해결 (질문되지 않음)
```

**활용 방식:**

1. **정밀한 역량 매핑**: 직무 요구사항(JD)과 지원자 역량을 노드 단위로 매핑
2. **지식 축적**: 면접 데이터가 쌓일수록 "프로젝트 관리자 채용 시 가장 중요한 질문 Top 5" 같은 인사이트 도출
3. **학습하는 시스템**: 성공한 채용의 면접 패턴 vs 실패한 채용의 패턴을 학습하여 더 나은 질문 제안
4. **조직 채용 역량 강화**: 시니어 면접관의 질문 패턴을 학습하여 주니어 면접관에게 코칭

**MVP vs 향후:**
- MVP: 기본 온톨로지 구조 구축 (역량-질문-답변 관계)
- Phase 2: 온톨로지 기반 추론 엔진 (자동 역량 갭 감지, 예측 모델)
- Phase 3: 조직별 커스텀 온톨로지 (회사마다 중요한 역량 다름)

**A4.** MVP에서는:
- Zoom (최우선 지원)
- Google Meet

Phase 2에서 추가 예정:
- Microsoft Teams

오프라인 면접의 경우 모바일 앱을 통한 녹음 지원도 Phase 2에서 검토 중입니다.

#### Q5. AI 요약의 정확도는 어느 정도인가요?

**A5.** 파일럿 테스트 결과:

- **전사 정확도**:
  - 한국어: 92-95% (OpenAI Whisper 기준)
  - 영어: 96-98%
  - 기술 용어/회사 고유 용어: 87-90% (커스텀 사전으로 개선 가능)

- **요약 품질** (면접관 평가):
  - "핵심 내용을 잘 포착했다": 89%
  - "내가 쓴 메모보다 완전하다": 94%
  - "요약만으로 평가 가능하다": 76%

- **질문 제안 유용성** (면접관 채택률):
  - 제안 질문을 실제로 사용: 67%
  - "도움이 되었다": 81%

**품질 보장 방법:**
1. 면접관이 AI 요약을 **항상 검토/수정** 가능 (Human-in-the-loop)
2. 원본 녹음/전사를 함께 제공하여 검증 가능
3. 피드백 루프: 면접관의 수정 내용을 학습하여 지속적으로 개선

#### Q6. 디브리프(상대평가) 기능과 어떻게 통합되나요?

**A6.** AI 면접 노트테이커와 디브리프 기능은 강력한 시너지를 발휘합니다:

**Before (디브리프만 사용):**
- 면접관들이 주관적 기억에 의존한 평가를 입력
- 디브리프 미팅에서 "그때 그 지원자가 뭐라고 했더라?" 같은 불확실성
- 평가 근거가 불명확하여 합의 어려움

**After (AI 노트테이커 + 디브리프):**
- 객관적 면접 기록을 기반으로 한 정확한 평가 입력
- 디브리프 미팅에서 실제 면접 녹음/전사 함께 검토
- "김철수 지원자는 리더십 질문에 대한 답변에서 구체적 사례를 3개 제시했지만, 이준호 지원자는 1개만 제시했다" 같은 증거 기반 비교
- 합의 시간 **추가로 30% 단축** (기존 디브리프 50% + AI 노트테이커 30% = 총 65% 단축)

**통합 워크플로우:**
1. 면접 진행 → AI 자동 녹음/전사
2. 면접 종료 → AI 요약 리포트 생성
3. 면접관 피드백 작성 → AI 요약을 참고하여 빠르게 완성
4. 디브리프 미팅 → 지원자별 면접 녹음/요약을 함께 보며 상대평가
5. 최종 합의 → 증거 기반 객관적 의사결정

### 사용자 경험 및 워크플로우

#### Q7. 지원자 동의는 어떻게 받나요?

**A7.** 지원자 동의 프로세스는 법적 컴플라이언스와 사용자 경험을 모두 고려하여 설계되었습니다:

**워크플로우:**

1. **면접 일정 생성 시 자동 포함**:
   - 채용 담당자가 라운드에서 면접 일정을 생성하면 자동으로 동의 안내 포함
   - 지원자에게 발송되는 캘린더 초대에 "이 면접은 품질 향상을 위해 녹음됩니다" 안내

2. **면접 시작 전 명시적 동의**:
   - AI 노트테이커가 화상회의에 참여하면 음성 안내: "이 면접은 채용 평가 목적으로 녹음됩니다. 동의하시면 '동의합니다'라고 말씀해주세요"
   - 채팅으로도 동의 링크 발송
   - 동의하지 않으면 녹음 시작 안 됨

3. **동의 기록 보관**:
   - 음성 동의 또는 채팅 클릭 기록을 법적 증거로 보관
   - 지원자는 언제든지 녹음 삭제 요청 가능

**동의 거부 시:**
- AI 노트테이커는 참여하지 않음
- 면접관이 수동으로 메모 작성 (기존 방식)
- 지원자에게 불이익 없음 (평가와 무관)

**개인정보보호:**
- 한국 개인정보보호법 완전 준수
- GDPR 준수 (EU 지원자)
- 녹음 데이터 암호화 저장
- 채용 프로세스 종료 후 자동 삭제 옵션 (설정 가능)

**A8.** 네, **AI는 조력자이며 최종 결정은 항상 면접관**이 합니다:

**수정 가능 범위:**
1. AI 요약 내용 편집 (추가/삭제/수정)
2. 잘못된 전사 수정 (예: 기술 용어 오류)
3. 평가 점수 조정
4. 하이라이트 태깅 (중요한 순간 표시)

**원본 보존:**
- 면접관의 수정본과 AI 원본을 모두 보관
- 수정 이력 추적 가능
- 디브리프 시 원본 녹음/전사도 함께 제공

**학습 루프:**
- 면접관의 수정 패턴을 학습하여 AI 품질 지속 개선

#### Q10. 면접관 편향을 어떻게 방지하나요?

**A10.** AI 면접 노트테이커는 편향을 감지하고 완화하는 여러 메커니즘을 포함합니다:

**MVP (Phase 1):**

1. **객관적 기록**: 면접관의 주관적 기억이 아닌 실제 대화 내용 기반 평가
2. **구조화된 평가**: AI가 직무 요구사항 대비 체크리스트 제공하여 놓친 역량 확인
3. **증거 요구**: "리더십이 부족하다"는 주관적 평가 시 "어떤 근거로?" 물어보고 면접 내용에서 관련 구간 제시

**Phase 2 (향후):**

1. **편향 패턴 감지**:
   - 특정 면접관이 특정 배경(학력, 성별, 나이 등)의 지원자를 일관되게 낮게 평가하는지 분석
   - "이 면접관은 여성 지원자의 리더십을 평균 1.2점 낮게 평가하는 경향" 같은 인사이트

2. **실시간 코칭**:
   - "이 질문은 나이 차별로 해석될 수 있습니다" 알림
   - "직무와 무관한 질문입니다" 경고

3. **Blind Review 옵션**:
   - 디브리프 시 지원자 이름/사진/배경 정보 숨기고 면접 내용만 검토
   - 순수하게 역량 기반 평가 유도

**주의사항:**
- AI도 편향을 가질 수 있음 (학습 데이터의 편향)
- 정기적으로 AI 평가 결과를 감사하여 편향 모니터링
- 최종 결정은 항상 인간(면접관, 채용 담당자)이 수행

### 기술 및 구현 관련

#### Q14. OpenAI Whisper를 선택한 이유는 무엇인가요?

**A14.** STT(Speech-to-Text) 기술 선택 시 여러 옵션을 비교 평가했습니다:

| 옵션 | 장점 | 단점 | 선택 여부 |
|------|------|------|----------|
| Google Speech-to-Text | 한국어 지원 우수, GCP 통합 | 비용 높음, 커스터마이징 제한 | ✗ |
| AWS Transcribe | AWS 인프라와 통합 용이 | 한국어 정확도 낮음 | ✗ |
| OpenAI Whisper | 다국어 정확도 최고, 오픈소스 | 자체 호스팅 필요 | ✓ |
| Clova Speech | 네이버, 한국어 특화 | 기술 용어 약함, API 제한 | Phase 2 검토 |

**OpenAI Whisper 선택 이유:**
1. **정확도**: 한국어/영어 모두 높은 정확도 (92-98%)
2. **비용 효율**: 자체 호스팅 시 API 비용 절감
3. **커스터마이징**: 회사별 기술 용어, 고유 명사 학습 가능
4. **다국어**: 향후 글로벌 확장 시 유리

**하이브리드 접근 (Phase 2):**
- 일반 대화: Whisper
- 한국 기업 특화 용어: Clova Speech 보조
- 고객이 선택 가능하도록 옵션 제공

#### Q15. 실시간 처리 vs 후처리 중 어느 것을 선택했나요?

**A15.** **하이브리드 접근**을 선택했습니다:

**실시간 처리 (면접 중):**
- 전사 (Transcription): 5초 지연 이내
- 추가 질문 제안: 지원자 답변 종료 후 10초 이내
- Slack 알림: 중요 순간 감지 시 즉시

**후처리 (면접 후):**
- AI 요약 리포트: 면접 종료 후 2-3분
- 온톨로지 구조 업데이트: 백그라운드 (5-10분)
- 디브리프 자료 생성: 모든 면접 완료 후

**이유:**
- 면접 중에는 **속도**가 중요 → 실시간 처리 (정확도 90%도 OK)
- 면접 후에는 **정확도**가 중요 → 후처리로 품질 향상 (95%+)
- 사용자 경험: 면접 끝나자마자 요약 확인 가능하지만, 더 정확한 최종 버전은 몇 분 후 제공

**성능 목표:**

### 핵심 가설 검증

이 기능은 다음 가설들을 검증합니다:

1. **H1**: 면접관이 필기 부담에서 해방되면 지원자와의 대화에 더 집중할 수 있고, 이는 면접 품질 향상으로 이어진다.
   - **측정**: 면접관 만족도, 지원자 경험 점수, 채용 품질

2. **H2**: 객관적 면접 기록은 주관적 기억보다 정확한 평가를 가능하게 한다.
   - **측정**: 디브리프 완료 시간, 평가 일관성, 채용 품질

3. **H3**: AI 기반 실시간 질문 제안은 면접관의 역량과 무관하게 일관된 고품질 면접을 가능하게 한다.
   - **측정**: 주니어 vs 시니어 면접관 평가 격차 감소, 면접관 역량 향상

4. **H4**: 면접 대화 데이터(동적 데이터)를 구조화하면 채용 검증의 질이 향상되고, 이는 장기적으로 조직의 채용 역량이 된다.
   - **측정**: 온톨로지 활용률, 채용 성공 예측 정확도 (Phase 2)
