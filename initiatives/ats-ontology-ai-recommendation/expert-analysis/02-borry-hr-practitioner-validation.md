# 보리(Borry) - HR 실무자 관점 검증

**전문가**: 보리 (10년 HR 실무 경험)
**분석일**: 2025-12-03
**관점**: HR 실무, 채용 현장, 사용자 니즈

---

## 요약 (Executive Summary)

이 아이디어는 기술적으로 흥미롭지만, **현장 도입 관점에서는 상당한 현실성 검증이 필요**합니다.

**핵심 메시지**:
- ✅ 동적 데이터는 실제로 활용하고 있지만, **PM이 생각하는 방식과는 매우 다름**
- ⚠️ 데이터 품질이 형편없음 (특히 대기업)
- ⚠️ AI 신뢰는 **조건부**: 설명 가능성, 오버라이드 가능성, 검증된 정확도 필수
- ✅ **위험 시그널 알림**이 1순위 (Bad Hire 비용이 연봉의 1.5-2배)

## 1. 현실성 평가: 데이터 활용 실태

### 실제로 이런 데이터들을 활용하고 있나요?

**짧은 답**: 네, 활용하고 있습니다. 하지만 **매우 다른 방식**으로.

#### 조직 규모별 현실

**스타트업/중소기업 (50명 이하)**
- 동적 데이터는 대부분 **채용 담당자의 머릿속**에만 존재
- "지난번 백엔드 개발자 A후보, 3라운드까지 가서 거절... 이번에 다시 연락?"
- "B회사 출신은 우리랑 문화가 안 맞더라. 면접까지 가도 결국 오퍼레터 안 받음"
- 이런 인사이트가 **Notion, Slack, 이메일, 개인 메모**에 파편화
- ATS에 체계적으로 입력? **절대 안 합니다. 시간이 없어요.**

**중견기업 (50-500명)** ← **당신의 타겟**
- 데이터는 있지만 **활용은 못함**
- 엑셀, 자체 ATS, Greenhouse/Lever 등에 산재
- "개발자 채용 평균 리드타임 60일인데 왜 그런지는 모름"
- 분석 시도는 하지만 **분기별 리포트 뽑는 게 전부**
- 실시간 의사결정 활용? 거의 불가능

**대기업 (500명 이상)**
- 데이터는 엄청나게 쌓이지만 **사일로화**
- Workday, SAP SuccessFactors + 자체 ATS
- **심각한 문제: 데이터 품질**
  - 면접관들이 평가 양식을 성의없이 채움 ("Good", "적합", "Pass")
  - 면접 스크립트? 녹취록? **거의 없음** (컴플라이언스 이슈)
  - 오퍼레터 사유는 "연봉", "다른 회사 선택" 체크만
- 컴플라이언스/개인정보보호 때문에 **데이터 접근 자체가 제한적**

### 어떤 상황에서 가장 필요할까?

**실제로 절박한 순간들**:

1. **긴급 백필**: "팀장 갑자기 퇴사, 3개월 안에 못 구하면 프로젝트 망함"
2. **동일 후보자 재검토**: "6개월 전 시니어로 지원했다가 떨어진 후보가 미드로 재지원"
3. **면접관 캘리브레이션**: "김 부장님은 항상 박하게 평가하는데, 실제 성과는 괜찮았음"

## 2. 가장 큰 Pain Point

### 정적 데이터 매칭의 한계

**실무에서 매일 겪는 문제**:

**문제 1: 맥락 부재**
- 이력서: "A회사 시니어 개발자, 3년"
- 실제 면접 후: "A회사가 5명 스타트업, 혼자 다 했음. 팀플레이 전무"
- ATS 검색으로 **절대 걸러낼 수 없음**

**문제 2: 타이밍 문제**
- 완벽한 후보인데 3개월 전 다른 회사 이직
- "6개월 후 재컨택" 메모 → **6개월 후 아무도 기억 못함**

**문제 3: 암묵적 레드플래그**
- "이 후보 3번 면접, 매번 연봉 협상에서 중단"
- 이런 패턴은 **수동 교차 분석**해야 보임

### 동적 데이터 활용으로 해결되는 것

**정말 해결되는 것**:
- 후보자 여정 파악: "왜 이 단계에서 이탈?" 패턴 분석
- 채용 병목 발견: "개발자 채용은 항상 CTO 면접에서 2주 지연"
- 오퍼레터 예측: "이 프로필은 과거 오퍼레터 후보와 유사"

**하지만 전제조건**:
- 데이터가 **정확하게 입력**되어야 함
- 데이터 입력이 **업무 부담 증가시키지 않아야**
- 데이터가 **법적 리스크 만들지 않아야**

## 3. AI 추천/알림 수용성

### HR 담당자들이 AI 제안을 신뢰할까?

**솔직한 답변**: **절대 맹신하지 않습니다.** 조건부로 참고만.

**실제 경험한 시나리오**:

**LinkedIn Recruiter "Recommended matches"**:
- 사용은 함. 하지만 **첫 화면으로만**
- 추천 이유 보고 "아, 이 키워드 때문이구나" 파악
- 최종 결정은 **내 판단**

**HireVue 같은 AI 면접 평가**:
- 대기업 도입했다가 **엄청난 반발**
- "AI가 후보의 뭘 알겠어? 표정 분석? 말도 안 돼"
- 결국 참고용으로만, 최종 결정권은 사람

**Greenhouse Predictive Scoring**:
- 과거 채용 데이터로 후보 점수 부여
- 문제: **과거 데이터가 편향되면** AI도 편향
- 특정 대학, 특정 회사 출신에 높은 점수 → DE&I 이슈

### 신뢰 조건 (5가지)

**이것들이 보장되어야 사용**:

1. **설명 가능성 (Explainability)**
   - "왜 이 후보를 추천?"이 **명확한 근거**와 함께
   - "과거 데이터 분석 결과" 같은 뭉뚱그린 설명 **신뢰 못함**
   - 예: "X회사 3년 + Y스킬 + 과거 유사 후보 5명과 90% 일치" ← 이 정도

2. **오버라이드 가능성**
   - AI 제안을 **무시하거나 수정** 가능해야
   - AI가 "탈락"이라 해도 내가 진행 가능해야

3. **검증된 정확도**
   - 최소 3-6개월 실사용 후 "AI 추천 vs 일반" 비교 데이터
   - 업계 벤치마크, 레퍼런스 고객 사례

4. **컴플라이언스 보장**
   - EEOC, GDPR 등 차별금지법 준수 증명
   - "AI가 나이, 성별, 인종으로 차별 안 한다" 감사 리포트

5. **점진적 도입**
   - 처음부터 전면 도입 **거부감**
   - "추천" → 신뢰 쌓이면 "알림" → 그 다음 "자동화"

## 4. 우선순위 검증

### 제안된 4가지 가치 순위 (실무 관점)

#### 🥇 **1순위: 위험 시그널 알림**

**왜?**
- 채용 실패(Bad Hire) 비용이 **엄청 큼**: 연봉의 1.5-2배
  - 재채용 비용, 팀 생산성 손실, 온보딩 시간 낭비
- 가장 두려운 순간: "이 후보 괜찮은데... 뭔가 놓친 게 있을까?"

**구체적 사례**:
- "이 후보는 과거 3번 면접, 모두 레퍼런스 체크 단계에서 중단"
- "유사 배경 후보 중 70%가 입사 6개월 이내 퇴사"
- "연봉 요구가 시장 평균 대비 40% 높음. 과거 이런 경우 오퍼레터율 85%"

#### 🥈 **2순위: 다음 액션 제안**

**왜?**
- HR 담당자는 **멀티태스킹**으로 정신없음
- 10개 포지션, 각 20-30명 후보 = 200-300개 태스크
- 놓치는 게 너무 많음

**유용한 제안**:
- "후보 A 마지막 연락 후 5일 경과. 팔로업 권장"
- "면접 피드백 미제출: 김 부장(3건), 이 과장(2건) → 리마인더"
- "최종 통과 후보 3명, 오퍼 전 연봉 밴드 검토 필요"

#### 🥉 **3순위: 프로액티브 인재 추천**

**왜?**
- 좋긴 한데, **신뢰 구축 필요**
- 초기 추천 품질 낮으면 (잘못된 추천 3번 받으면) 기능 안 씀
- 단, 제대로 작동하면 **게임체인저**

**조건**:
- 추천 정확도 80% 이상
- 추천 이유 명확
- "지금 당장" vs "6개월 후" 컨택 구분

#### 4순위: 인재 검증 정확도 향상

**왜?**
- 이건 **백엔드 개선**처럼 느껴짐
- HR은 "검증 정확도" 개념 자체를 잘 모름
- 대신 "추천", "알림", "제안"의 **품질**로 간접 체감
- 단독 기능으로 제시하면 "그래서 뭐가 좋아지는데?" 반응

## 5. 실무에서 가장 자주 발생하는 문제 TOP 5

1. **후보자 응답률 저조** (가장 큰 문제)
   - 100명 컨택해서 5명만 응답
   - 어떤 메시지, 타이밍에 보내야 응답률 높은지 모름

2. **면접 스케줄링 지옥**
   - 후보 3명 × 면접관 4명 = 12회 왕복 이메일
   - Calendly 써도 한계

3. **면접관 피드백 누락/지연**
   - 면접 끝나고 피드백 안 주면 3일 대기
   - 후보는 기다리다 다른 오퍼 받음

4. **오퍼레터 사유 불명확**
   - "다른 회사 선택" → 왜? 연봉? 브랜드? 업무?
   - 패턴 분석 못하면 계속 같은 실수

5. **채용 프로세스 병목**
   - 어느 단계가 느린지 모름
   - "개발자 채용이 왜 오래 걸려?" → 데이터로 답 못함

## 6. 조직 규모별/채용 단계별 차이

### 스타트업 (50명 이하)

**특징**: HR 1명, 속도 최우선, ATS 사용률 낮음

**니즈**: "복잡한 기능 NO. 빠르게 좋은 후보 찾기"

**수용성**:
- 새 툴 도입 **매우 빠름** (결정권자 1명)
- **비용 민감** (월 $100 이상 부담)
- 온톨로지, 지식그래프 = **오버엔지니어링**으로 느껴짐

### 중견기업 (50-500명) ← **타겟**

**특징**: HR 팀 3-10명, ATS 사용 중, 효율성 필요

**니즈**:
- 데이터 기반 의사결정 원함
- 하지만 **데이터 분석 역량 부족**
- "우리 채용 프로세스 어디가 문제?" 알고 싶음

**수용성**:
- 신기술 도입 **신중** (ROI 증명 필요)
- 파일럿 선호 (3-6개월)
- 성공 사례 필수 (레퍼런스 2-3개)

**가장 유용한 기능**:
- 위험 시그널 (Bad Hire 방지)
- 병목 분석 (리드타임 단축)
- 면접관 성과 분석

### 대기업 (500명 이상)

**특징**: HR 수십 명, 엔터프라이즈 HRIS, 컴플라이언스 최우선

**니즈**: 효율성보다 **리스크 관리**, DE&I 메트릭, 감사 대응

**수용성**:
- 신기술 도입 **매우 느림** (12-24개월)
- 보안/프라이버시 감사 필수
- 기존 시스템 통합 필수 (Workday API)
- POC → 파일럿 → 롤아웃 (최소 2년)

**AI 수용 장벽**:
- "AI가 차별하지 않는다는 걸 어떻게 증명?"
- "후보자가 AI 평가에 이의제기하면?"
- "GDPR/CCPA 준수는? 데이터 삭제 요청은?"

### 채용 단계별 유용성

| 단계 | 현재 문제 | AI 가치 | 채택 가능성 |
|------|----------|---------|------------|
| Sourcing | 수백 명 중 누구부터? | 우선순위 스코어링 | ⭐⭐⭐⭐⭐ |
| Screening | 이력서 100장 반나절 | 자동 스크리닝 | ⭐⭐⭐ |
| 면접 후 평가 | 주관적, 일관성 없음 | 캘리브레이션 | ⭐⭐ (민감) |
| 오퍼 단계 | 오퍼 내고 거절당함 | 리스크 예측 | ⭐⭐⭐⭐ |
| 온보딩 후 | 3개월 만에 퇴사 | Early Warning | ⭐⭐⭐ |

## 7. 실무자의 솔직한 조언

### 이 제품이 성공하려면

#### 1. 데이터 입력 부담을 없애라
- "더 많은 데이터 입력하세요" → **절대 안 통함**
- 자동화 필수: 이메일, 캘린더, Slack 연동
- 면접 녹취 → 자동 요약 → 구조화된 피드백

#### 2. Quick Win을 먼저 보여줘라
- 온톨로지, 지식그래프는 엔지니어에겐 섹시, HR에겐 **의미 없음**
- "이 기능 쓰면 응답률 2배" ← 이게 먹힘
- 첫 30일 안에 **가시적 성과** (안 그러면 이탈)

#### 3. 기존 워크플로우를 바꾸지 마라
- "우리 ATS로 갈아타세요" → 실패 90%
- "기존 ATS + 플러그인" → 성공률 높음
- Greenhouse/Lever/Workday API 연동 필수

#### 4. 신뢰를 점진적으로
- Phase 1: 인사이트 ("이런 패턴 보임")
- Phase 2: 추천 ("이 후보 보세요")
- Phase 3: 자동화 ("자동으로 할까요?")
- **처음부터 자동화 NO**

#### 5. ROI를 명확히
- "AI 기술 좋아요" ← 안 먹힘
- "리드타임 30% 단축, Bad Hire 50% 감소, $50K 절감" ← 먹힘
- 파일럿 2-3개로 **정량 데이터** 확보

### 제품 개발 순서 제안

**MVP (바로 돈 낼 의향 있음)**:
1. 기존 ATS 연동 (Greenhouse API)
2. 오퍼레터 리스크 알림
3. 다음 액션 리마인더

**V2**:
4. 후보자 우선순위 스코어링
5. 병목 분석 대시보드
6. 면접관 피드백 자동 수집

**V3 (온톨로지 본격)**:
7. 프로액티브 추천
8. 지식그래프 인사이트
9. 예측 분석

## 8. 가장 중요한 것

### HR 담당자는 "AI 기술"에 돈 내지 않습니다.
### "채용 성공률 향상"에 돈 냅니다.

온톨로지가 아무리 정교해도, **"다음 주에 급하게 뽑아야 하는 시니어 개발자 찾는 데 도움 안 되면"** 안 씁니다.

대신, 단순한 알림이라도 **"오늘 팔로업할 후보 5명 놓칠 뻔했는데 알림 받아서 연락, 그 중 1명 면접 잡힘"**이면 기꺼이 돈 냅니다.

**기술 중심이 아니라 문제 해결 중심**으로 접근하세요.

## 구체적 검증 액션

1. **실제 HR 10명과 인터뷰** (내 의견은 내 경험일 뿐)
