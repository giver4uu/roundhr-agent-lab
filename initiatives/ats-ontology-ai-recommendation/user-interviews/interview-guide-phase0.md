# Phase 0 고객 인터뷰 가이드

**목적**: 가정 1, 2, 4 검증 (가치, 사용성, 경제성)
**대상**: 10명 (현재 5 + 잠재 3 + churned 2)
**기간**: Week 2 (2025-12-09 ~ 12-13)
**방법**: 1:1 인터뷰 (45-60분)
**프레임워크**: JTBD (Jobs to be Done) + Wizard of Oz

---

## 인터뷰 목표 및 성공 기준

### 검증할 가정

**가정 1: 가치 가정** (최고 리스크)
- 진술: "채용 담당자는 프로액티브한 AI 인사이트를 가치있게 여겨 행동을 바꿀 것이다"
- 성공 기준: **60% 이상**이 Wizard of Oz 시나리오에 "이거 나오면 사용하겠다" + 구체적 액션 설명

**가정 2: 사용성 가정** (높은 리스크)
- 진술: "리크루터가 현재 워크플로우를 바꾸지 않고도 AI 기능을 통합할 수 있다"
- 성공 기준: **70% 이상**이 "주간 리포트 이메일"이나 "Slack 알림" 형태에 긍정 반응

**가정 4: 경제성 가정**
- 진술: "고객이 기존 ATS 비용의 30-50% 프리미엄을 지불할 의향이 있다"
- 성공 기준: **40% 이상**이 목표 가격 범위($X-Y/user/month) 수용

### 학습 목표

1. **Pain Point 패턴**: 3개 이상 명확한 공통 Pain Point 발견
2. **신뢰 조건**: AI 인사이트를 신뢰하기 위한 조건 (설명 가능성, 정확도, 오버라이드)
3. **행동 유발 인사이트**: 어떤 인사이트가 실제 액션을 유발하는가?
4. **우선순위 검증**: 병목 진단 vs 위험 시그널 vs 액션 리마인더 중 가장 시급한 것?

## 인터뷰 스크립트

### Part 0: 도입 (5분)

**목표**: 라포 형성, 인터뷰 목적 설명, 동의 획득

```
안녕하세요 {이름}님, 시간 내주셔서 감사합니다!

오늘 인터뷰는 채용 담당자분들의 실제 Pain Point를 이해하고,
저희가 기획 중인 새로운 기능이 정말 도움이 될지 검증하기 위한 자리입니다.

정답은 없으니 편하게 솔직한 의견 부탁드립니다.
특히 "이거 별로인 것 같은데..." 같은 부정적 피드백이 가장 중요합니다!

인터뷰 내용을 녹음해도 될까요? (동의 확인)
녹음은 나중에 정확한 기록을 위한 것이고, 외부에 공유되지 않습니다.

시작하기 전에 질문 있으신가요?

좋습니다, 그럼 시작하겠습니다!
```

### Part 1: 배경 파악 (5분)

**목표**: 인터뷰 대상자의 맥락 이해

```
Q1. 먼저 간단히 자기소개 부탁드립니다.
- 회사 규모는 어떻게 되나요?
- HR 팀은 몇 명인가요?
- {이름}님의 역할은 구체적으로 무엇인가요?

Q2. 채용 프로세스는 어떻게 진행되나요?
- 한 달에 평균 몇 명 정도 채용하시나요?
- 어떤 직군을 주로 채용하시나요? (개발, 영업, 기획 등)
- 평균 채용 기간은 얼마나 걸리나요?

Q3. 현재 어떤 도구를 사용하시나요?
- ATS는 무엇을 쓰시나요? (Greenhouse, Lever, 자체 솔루션 등)
- 다른 도구도 함께 사용하나요? (Notion, Slack, 엑셀 등)
- 만족도는 어떤가요? (1-10점)
```

**노트 포인트**:
- 회사 규모, HR 팀 크기, 채용 볼륨 → 페르소나 분류용
- 사용 중인 ATS → 경쟁 비교, 통합 가능성
- 만족도 → churned 고객의 경우 이탈 이유 파악

### Part 2: JTBD - 최근 채용 실패/지연 사례 (10분)

**목표**: 구체적인 Pain Point 발견, "왜 실패했는가?" 이해

```
Q4. 최근 6개월 간 가장 힘들었던 채용 케이스를 떠올려주세요.
- 어떤 포지션이었나요?
- 무엇이 잘못되었나요?
- 구체적으로 어떤 상황이었는지 자세히 말씀해주세요.

[따라가는 질문들]
- 그때 어떤 시그널을 놓쳤다고 생각하시나요?
- 만약 ____ 정보가 있었다면 다른 결정을 하셨을까요?
- 그 케이스에서 가장 답답했던 순간은 언제였나요?
- 결과적으로 어떻게 해결하셨나요? (또는 해결 못했나요?)

Q5. 채용이 예상보다 오래 걸린 경우가 있었나요?
- 평소에는 몇 주인데, 그때는 얼마나 걸렸나요?
- 어느 단계에서 가장 오래 걸렸나요?
- 왜 그렇게 오래 걸렸다고 생각하시나요?
- 그걸 미리 알았다면 어떻게 하셨을까요?

Q6. 채용했는데 조기 퇴사한 케이스가 있었나요?
- 입사 후 얼마 만에 퇴사했나요?
- 왜 퇴사했다고 생각하시나요?
- 면접 과정에서 놓친 레드플래그가 있었나요?
- 다시 돌아간다면 어떤 질문을 더 했을까요?
```

**노트 포인트**:
- **구체적 행동**에 집중 (일반화된 의견 X)
- "시그널을 놓쳤다" → 어떤 데이터가 있었으면 좋았을까?
- "오래 걸렸다" → 병목 진단 Use Case 검증
- "조기 퇴사" → 위험 시그널 Use Case 검증

### Part 3: 현재 데이터 활용 실태 (10분)

**목표**: 데이터 품질 간접 검증, 활용 패턴 이해

```
Q7. 현재 채용 데이터를 어떻게 활용하시나요?
- 면접 후 평가 내역은 어떻게 관리하시나요?
- ATS에 꼼꼼히 기록하시나요, 아니면 메모/이메일로만?
- 평가 양식은 있나요? (구조화 vs 프리텍스트)

Q8. 과거 채용 데이터를 다시 찾아본 적 있나요?
- 어떤 상황에서요?
- 찾기 쉬웠나요, 어려웠나요?
- 예시: "6개월 전에 지원했던 후보가 다시 지원했을 때"

Q9. 오퍼레터한 후보가 있었나요?
- 오퍼레터 사유를 어떻게 기록하시나요?
- 그 데이터를 나중에 활용한 적 있나요? (예: 패턴 분석)

Q10. 채용 프로세스 분석을 해본 적 있나요?
- 예: "평균 리드타임", "단계별 전환율", "면접관별 평가 경향"
- 분석하셨다면 어떻게 하셨나요? (엑셀 수동? ATS 리포트?)
- 분석 결과를 실제 개선에 활용하셨나요?
```

**노트 포인트**:
- 데이터 입력 습관 → 품질 예측
- "과거 데이터를 찾기 어렵다" → Use Case 검증
- "분석 안 해봤다" → 데이터 활용 역량 낮음 → PM 도구 필요성 높음
