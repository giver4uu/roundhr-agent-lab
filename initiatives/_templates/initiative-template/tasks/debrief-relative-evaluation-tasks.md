# 디브리프(상대평가) 기능 개발 태스크 리스트

**PRD 문서**: `prd-debrief-relative-evaluation.md`  
**프로젝트**: 디브리프(상대평가) 기능  
**작성일**: 2024년 12월 19일  
**상태**: 진행 중

---

## 프로젝트 개요

디브리프(상대평가) 기능은 라운드의 핵심 미션인 "적합한 인재 채용"을 지원하는 핵심 기능입니다. 면접관들이 평가를 종합하고 지원자들을 상대적으로 비교할 수 있는 플랫폼을 제공하여 채용 과정의 효율성과 투명성을 높이는 것을 목표로 합니다.

---

## 주요 솔루션

### 솔루션 1: 스코어카드 기반 직관적 비교 UI + 가중치 적용
- 평가 항목별로 여러 지원자 점수를 한 번에 비교
- 면접관별 전문성에 따른 가중치 자동 적용
- 직관적이고 객관적인 상대평가 지원

### 솔루션 2: 통합 정보 대시보드 + AI 보조 기능
- 평가 내역 + 추가 정보를 모두가 같은 화면에서 확인
- AI가 평가 일관성 체크 및 요약 리포트 제공
- 실시간 협업과 의사결정 지원

### 솔루션 3: 블라인드 평가 모드 + 투표 기반 합의 시스템
- 면접관들이 독립적으로 평가 후 다른 평가 확인
- 각 평가 항목별로 투표하여 합의점 도출
- 편견 최소화와 객관적 의사결정 지원

## 개발 태스크

### Phase 1: 핵심 평가 종합 기능 (4주)

#### T1: 평가 데이터 모델 설계 및 구현
- [ ] **T1.1**: 평가 항목 스키마 설계 (기술역량, 문화적합성, 리더십 등)
- [ ] **T1.2**: 면접관별 평가 데이터 모델 구현
- [ ] **T1.3**: 지원자별 평가 집계 로직 구현
- [ ] **T1.4**: 평가 데이터 검증 및 무결성 보장 로직 구현

#### T2: 스코어카드 기반 비교 UI 개발
- [ ] **T2.1**: 평가 항목별 지원자 점수 표시 테이블 컴포넌트 개발
- [ ] **T2.2**: 지원자 선택 및 비교 기능 구현
- [ ] **T2.3**: 점수 시각화 (막대그래프, 히트맵) 컴포넌트 개발
- [ ] **T2.4**: 반응형 디자인 적용 (모바일/태블릿 지원)

#### T3: 가중치 적용 시스템 구현
- [ ] **T3.1**: 면접관별 전문성 가중치 설정 기능
- [ ] **T3.2**: 가중치 기반 종합 점수 계산 로직 구현
- [ ] **T3.3**: 가중치 적용 결과 시각화
- [ ] **T3.4**: 가중치 설정 권한 관리 (채용 담당자만 수정 가능)

### Phase 2: 통합 정보 대시보드 (3주)

#### T4: 통합 대시보드 UI 개발
- [ ] **T4.1**: 지원자 프로필 통합 뷰 컴포넌트 개발
- [ ] **T4.2**: 평가 내역 + 추가 정보 통합 표시
- [ ] **T4.3**: 실시간 동기화 기능 구현 (WebSocket)
- [ ] **T4.4**: 대시보드 레이아웃 및 네비게이션 구현

#### T5: AI 보조 기능 개발
- [ ] **T5.1**: 평가 일관성 체크 알고리즘 구현
- [ ] **T5.2**: 지원자별 강점/약점 자동 요약 기능
- [ ] **T5.3**: 평가 패턴 분석 및 인사이트 제공
- [ ] **T5.4**: AI 결과 시각화 및 설명 기능

### Phase 3: 블라인드 평가 및 합의 시스템 (3주)

#### T6: 블라인드 평가 모드 구현
- [ ] **T6.1**: 독립 평가 단계 UI 구현
- [ ] **T6.2**: 평가 완료 후 다른 평가 공개 기능
- [ ] **T6.3**: 평가 순서 및 타이밍 관리
- [ ] **T6.4**: 블라인드 평가 진행률 추적

#### T7: 투표 기반 합의 시스템
- [ ] **T7.1**: 평가 항목별 투표 기능 구현
- [ ] **T7.2**: 합의 도출 알고리즘 (다수결, 가중평균 등)
- [ ] **T7.3**: 투표 결과 시각화 및 합의 상태 표시
- [ ] **T7.4**: 합의 미달성 시 재투표 및 토론 기능

### Phase 4: 디브리프 프로세스 지원 (2주)

#### T8: 실시간 디브리프 지원
- [ ] **T8.1**: 실시간 토론 스레드 기능 구현
- [ ] **T8.2**: 면접관 온라인/오프라인 상태 표시
- [ ] **T8.3**: 디브리프 진행 상황 타임라인 구현
- [ ] **T8.4**: 디브리프 완료 자동 감지 및 알림

#### T9: 비동기 디브리프 지원
- [ ] **T9.1**: 단계별 디브리프 진행 상태 관리
- [ ] **T9.2**: 비동기 참여자 알림 시스템
- [ ] **T9.3**: 디브리프 템플릿 및 가이드 제공
- [ ] **T9.4**: 비동기 디브리프 완료율 추적

### Phase 5: 최종 결과 생성 및 통합 (2주)

#### T10: 최종 결과 생성 기능
- [ ] **T10.1**: 합격/불합격 결정 투표 기능
- [ ] **T10.2**: 최종 결정 근거 자동 정리 기능
- [ ] **T10.3**: 상세 평가 리포트 PDF 생성
- [ ] **T10.4**: 채용 프로세스 다음 단계 연동

#### T11: 시스템 통합 및 최적화
- [ ] **T11.1**: 기존 면접 평가 시스템과의 통합
- [ ] **T11.2**: 성능 최적화 (캐싱, 데이터베이스 쿼리 최적화)

## 성공 지표

### 사용자 효율성 지표
- 디브리프 완료 시간: 기존 대비 50% 단축 (목표: 2시간 → 1시간)
- 평가 일관성: 면접관 간 평가 점수 표준편차 20% 감소
- 사용자 만족도: 면접관 만족도 4.5/5.0 이상

### 비즈니스 품질 지표
- 채용 성공률: 디브리프를 거친 채용의 6개월 이직률 30% 감소
- 의사결정 정확도: 최종 합격자의 실제 성과 예측 정확도 80% 이상
- 프로세스 투명성: 평가 근거 문서화율 100%

### 경험 및 협업 지표
- 기능 활용률: 디브리프 기능 사용률 90% 이상
- 협업 효율성: 비동기 디브리프 완료율 95% 이상
- 학습 곡선: 신규 사용자의 기능 숙련도 달성 시간 50% 단축
